# ローカルLLM チャットシステム

このプロジェクトは、`llama.cpp` (`llama-server.exe`) をバックエンドとして利用し、ブラウザから快適にローカルLLMと対話するための高機能なWeb UIを提供します。

## 主な機能

- **直感的なチャットインターフェース**: 快適なレスポンスとリアルタイムの応答表示。
- **高度な生成パラメータ設定**: 
    - Temperature, Top P, Max Tokens, 各種 Penalty の調整。
    - システムプロンプトのプリセット（翻訳家、プログラマー、メイド等）とカスタム保存。
- **マルチモーダル・ファイル対応**:
    - 画像の解析（対応モデル使用時）。
    - PDF, テキスト, Markdownファイルの読み込み。
- **カスタマイズ**:
    - テーマカラーの変更。
    - 壁紙画像の設定（透明度、ぼかし、フィッティング調整）。
- **データ管理**:
    - ブラウザのローカルストレージへの履歴保存。
    - チャット履歴のJSONエクスポート/インポート。
    - ストレージ使用量の可視化。
- **スマートなロード**: サーバーの起動状況を監視し、準備が整い次第自動でチャット画面へ遷移。

## ファイル構成

- `llama-server.exe`: LLMサーバー本体（llama.cpp）。
- `models/`: GGUF形式のモデルファイルを格納。
- `custom_ui/`:
    - `index.html`: メインのチャット画面。
    - `loading.html`: サーバー起動待ち画面（`/health` エンドポイントを監視）。
    - `script.js`: チャットロジック、API連携、UI制御（モジュール形式）。
    - `style.css`: デザイン定義。
- `run.bat`: サーバーの起動とブラウザ表示を自動化するスクリプト。

## セットアップと実行方法

1. **準備**: 
   - `llama-server.exe` を本ディレクトリに配置してください。
   - `models/` ディレクトリに必要な GGUF モデルを配置してください。

2. **起動**:
   - `run.bat` を実行します。
   - サーバーがバックグラウンドで起動し、ブラウザが自動的に開きます。
   - サーバーの準備ができるまで「読み込み中」画面が表示され、完了するとチャット画面に切り替わります。

3. **サーバーの停止**:
   - `llama-server.exe` が動作しているコンソールウィンドウを閉じるか、UI上の「サーバーを停止」ボタン（環境による）を使用してください。

## 新しいモデルの追加手順

1. **モデルの入手**: Hugging Face などから、`.gguf` 形式のモデルファイルをダウンロードします。
2. **配置**: ダウンロードしたファイルを `models/` フォルダ内に保存します。
3. **スクリプトの更新**: `run.bat` を右クリックして「編集」を選択し、`-m` オプションのパスを新しいファイル名に書き換えます。
   ```batch
   start "" "llama-server.exe" -m models/あなたのモデル名.gguf --port 8080
   ```

## 主要な起動オプションの解説

パフォーマンスや動作を調整するために、`run.bat` 内で以下のオプションを組み合わせて使用できます。

- `-c <size>` (`--ctx-size`): コンテキストサイズを指定します（例: `-c 4096`）。大きくすると長い対話が可能になりますが、メモリ消費量が増えます。
- `-ngl <layers>` (`--n-gpu-layers`): GPUにオフロードするレイヤー数を指定します（例: `-ngl 35`）。
  - NVIDIAやMac (Metal) 環境で非常に有効です。
  - `99` などの大きな数値を指定すると、全レイヤーをGPUに乗せようと試みます（VRAM容量に依存）。
- `-t <threads>` (`--threads`): 使用するCPUスレッド数を指定します。
- `--flash-attn`: Flash Attention を有効にして、メモリ消費を抑えつつ推論を高速化します（対応環境のみ）。
- `--host <address>`: サーバーがリッスンするホスト名を指定します（デフォルトは `127.0.0.1`）。

**設定例:**
`start "" "llama-server.exe" -m models/model.gguf -c 8192 -ngl 99 --flash-attn`

## VRAM容量に応じた自動設定 (上級者向け)

PCのVRAM容量を自動判別して `-ngl` の値を最適化したい場合、`run.bat` を以下のように書き換えることで、環境に応じた動的な起動が可能になります。

```batch
@echo off
setlocal

rem VRAM容量(GB)を取得
for /f %%i in ('powershell -command "[math]::Round((Get-CimInstance Win32_VideoController | Measure-Object -Property AdapterRAM -Sum).Sum / 1GB)"') do set VRAM_GB=%%i

rem VRAM容量に応じたオフロードレイヤー数の決定
if %VRAM_GB% GEQ 12 (
    set NGL=99
) else if %VRAM_GB% GEQ 6 (
    set NGL=35
) else (
    set NGL=0
)

echo 検出VRAM: %VRAM_GB%GB / 設定NGL: %NGL%

start "" "llama-server.exe" -m models/Bonsai-1.7B-Q1_0.gguf --port 8080 -ngl %NGL%
```
※ `-ngl 99` は「可能な限り全てのレイヤーをGPUに送る」という意味になります。

## 注意事項

- **リソース消費**: 大規模な言語モデルを使用する場合、多くのシステムリソース（特にVRAMとRAM）を消費します。お使いのPCのスペックをご確認ください。
- **生成コンテンツの品質**: LLMが生成するコンテンツは、モデルの学習データに依存するため、不正確な情報や不適切な表現を含む可能性があります。利用者の判断でご使用ください。
- **開発状況**: 本プロジェクトは開発途上であり、予期せぬ動作やバグが含まれる可能性があります。
- **パフォーマンス調整**: `llama-server.exe` の起動オプションは、モデルやハードウェア環境によって最適な設定が異なります。パフォーマンスが出ない場合は、各種オプションを調整してください。
- **セキュリティ**: ローカル環境での利用を想定していますが、外部からのアクセスを許可する場合は、セキュリティ対策を適切に行ってください。
